<h2 class="anchor anchorTargetHideOnScrollNavbar_vjPI" id="adaptive-playwright-crawler">Adaptive Playwright crawler<a class="hash-link" href="https://crawlee.dev/blog/crawlee-for-python-v06#adaptive-playwright-crawler" title="Direct link to Adaptive Playwright crawler">​</a></h2>
<p>The new <a href="https://www.crawlee.dev/python/api/class/AdaptivePlaywrightCrawler" rel="noopener noreferrer" target="_blank"><code>AdaptivePlaywrightCrawler</code></a> is a hybrid solution that combines the best of two worlds: full browser rendering with <a href="https://www.playwright.dev/" rel="noopener noreferrer" target="_blank">Playwright</a> and lightweight HTTP-based crawling (using, for example, <a href="https://www.crawlee.dev/python/api/class/BeautifulSoupCrawler" rel="noopener noreferrer" target="_blank"><code>BeautifulSoupCrawler</code></a> or <a href="https://www.crawlee.dev/python/api/class/ParselCrawler" rel="noopener noreferrer" target="_blank"><code>ParselCrawler</code></a>). It automatically switches between the two methods based on real-time analysis of the target page, helping you achieve lower crawl costs and improved performance when crawling a variety of websites.</p>

<h2 class="anchor anchorTargetHideOnScrollNavbar_vjPI" id="browserforge-fingerprints">Browserforge fingerprints<a class="hash-link" href="https://crawlee.dev/blog/crawlee-for-python-v06#browserforge-fingerprints" title="Direct link to Browserforge fingerprints">​</a></h2>
<p>To help you avoid detection and blocking, Crawlee now integrates the <a href="https://www.github.com/daijro/browserforge" rel="noopener noreferrer" target="_blank">browserforge</a> library - intelligent browser header &amp; fingerprint generator. This feature simulates real browser behavior by automatically randomizing HTTP headers and fingerprints, making your crawling sessions significantly more resilient against anti-bot measures.</p>
<p>With <a href="https://www.github.com/daijro/browserforge" rel="noopener noreferrer" target="_blank">browserforge</a> fingerprints enabled by default, your crawler sends realistic HTTP headers and user-agent strings. HTTP-based crawlers, which use <a href="https://www.crawlee.dev/python/api/class/HttpxHttpClient" rel="noopener noreferrer" target="_blank"><code>HttpxHttpClient</code></a> by default benefit from these adjustments, while the <a href="https://www.crawlee.dev/python/api/class/CurlImpersonateHttpClient" rel="noopener noreferrer" target="_blank"><code>CurlImpersonateHttpClient</code></a> employs its own stealthy techniques. The <a href="https://www.crawlee.dev/python/docs/guides/playwright-crawler" rel="noopener noreferrer" target="_blank"><code>PlaywrightCrawler</code></a> adjusts HTTP headers and browser fingerprints accordingly. Together, these improvements make your crawlers much harder to detect.</p>

<h2 class="anchor anchorTargetHideOnScrollNavbar_vjPI" id="cli-dependencies">CLI dependencies<a class="hash-link" href="https://crawlee.dev/blog/crawlee-for-python-v06#cli-dependencies" title="Direct link to CLI dependencies">​</a></h2>
<p>In v0.6, we've reduced the size of the core package by moving CLI (template creation) dependencies to optional extras. This change reduces the package footprint, keeping the base installation lightweight. To use Crawlee's CLI for creating new projects, simply install the package with the CLI extras.</p>
