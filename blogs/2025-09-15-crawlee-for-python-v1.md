<p>We launched Crawlee for Python in beta mode in <a href="https://www.crawlee.dev/blog/launching-crawlee-python" rel="noopener noreferrer" target="_blank">July 2024</a>. Over the past year, we received many early adopters, tremendous interest in the library from the Python community, more than 6000 stars on GitHub, a dozen contributors, and many feature requests.</p>
<p>After months of development, polishing, and community feedback, the library is leaving beta and entering a production/stable development status.</p>
<p><strong>We are happy to announce Crawlee for Python v1.0.</strong></p>
<p>From now on, Crawlee for Python will strictly follow <a href="https://www.semver.org/" rel="noopener noreferrer" target="_blank">semantic versioning</a>. You can now rely on it as a stable foundation for your crawling and scraping projects, knowing that breaking changes will only occur in major releases.</p>

<h2 class="anchor anchorTargetHideOnScrollNavbar_vjPI" id="new-storage-client-system">New storage client system<a class="hash-link" href="https://crawlee.dev/blog/crawlee-for-python-v1#new-storage-client-system" title="Direct link to New storage client system">​</a></h2>
<p>One of the biggest architectural changes in Crawlee v1 is the introduction of a new storage client system. Until now, datasets, key–value stores, and request queues were handled in slightly different ways depending on where they were stored. With v1, this has been unified under a single, consistent interface.</p>
<p>This means that whether you're storing data in memory, on the local file system, in a database, on the Apify platform, or even using a custom backend, the API remains the same. The result is less duplication, better extensibility, and a cleaner developer experience. It also opens the door for the community to build and share their own storage client implementations.</p>

<h3 class="anchor anchorTargetHideOnScrollNavbar_vjPI" id="new-experimental-sql-storage-client">New experimental SQL storage client<a class="hash-link" href="https://crawlee.dev/blog/crawlee-for-python-v1#new-experimental-sql-storage-client" title="Direct link to New experimental SQL storage client">​</a></h3>
<p>Crawlee v1 introduces an experimental <a href="https://www.crawlee.dev/python/api/class/SqlStorageClient" rel="noopener noreferrer" target="_blank"><code>SqlStorageClient</code></a> that enables persistent storage using SQL databases. Currently, SQLite and PostgreSQL are supported. This storage backend supports concurrent access from multiple crawler processes, enabling distributed crawling scenarios.</p>
<p>The SQL storage client uses <a href="https://www.sqlalchemy.org/" rel="noopener noreferrer" target="_blank">SQLAlchemy 2+</a> under the hood, providing automatic schema creation, connection pooling, and database-specific optimizations. It maintains the same interface as other storage clients, making it easy to switch between different storage backends without changing your crawling logic.</p>

<h2 class="anchor anchorTargetHideOnScrollNavbar_vjPI" id="adaptive-playwright-crawler">Adaptive Playwright crawler<a class="hash-link" href="https://crawlee.dev/blog/crawlee-for-python-v1#adaptive-playwright-crawler" title="Direct link to Adaptive Playwright crawler">​</a></h2>
<p>Some websites can be scraped quickly with plain HTTP requests, while others require the full power of a browser to render dynamic content. Traditionally, you had to decide upfront whether to use one of the lightweight HTTP-based crawlers (<a href="https://www.crawlee.dev/python/api/class/ParselCrawler" rel="noopener noreferrer" target="_blank"><code>ParselCrawler</code></a> or <a href="https://www.crawlee.dev/python/api/class/BeautifulSoupCrawler" rel="noopener noreferrer" target="_blank"><code>BeautifulSoupCrawler</code></a>) or a browser-based <a href="https://www.crawlee.dev/python/api/class/PlaywrightCrawler" rel="noopener noreferrer" target="_blank"><code>PlaywrightCrawler</code></a>. Crawlee v1 introduces the <a href="https://www.crawlee.dev/python/api/class/AdaptivePlaywrightCrawler" rel="noopener noreferrer" target="_blank"><code>AdaptivePlaywrightCrawler</code></a>, which automatically chooses the right approach for each page.</p>
<p>The adaptive crawler uses a detection mechanism: it compares the results of plain HTTP requests with those of a browser-rendered version of the same page. If both match, it can continue with the faster HTTP approach; if differences appear, it falls back to browser-based crawling. Over time, it builds confidence about which rendering type is needed for different pages, occasionally re-checking with the browser to ensure its predictions stay correct.</p>
<p>This makes your crawls faster and cheaper, while still allowing you to reliably handle complex, dynamic websites. In practice, you get the best of both worlds: speed on simple pages and robustness on modern, JavaScript-heavy sites. For advanced options, such as customizing the detection strategy, see the <a href="https://www.crawlee.dev/python/docs/guides/adaptive-playwright-crawler" rel="noopener noreferrer" target="_blank">Adaptive Playwright crawler guide</a>.</p>

<h2 class="anchor anchorTargetHideOnScrollNavbar_vjPI" id="impit-http-client">Impit HTTP client<a class="hash-link" href="https://crawlee.dev/blog/crawlee-for-python-v1#impit-http-client" title="Direct link to Impit HTTP client">​</a></h2>
<p>Crawlee v1 introduces a brand-new default HTTP client: <a href="https://www.crawlee.dev/python/api/class/ImpitHttpClient" rel="noopener noreferrer" target="_blank"><code>ImpitHttpClient</code></a>, powered by the <a href="https://www.github.com/apify/impit" rel="noopener noreferrer" target="_blank">Impit</a> library. Impit is also developed as an open-source project by Apify, so you can dive into the internals or contribute improvements yourself.</p>

<h2 class="anchor anchorTargetHideOnScrollNavbar_vjPI" id="robots-exclusion-standard">Robots exclusion standard<a class="hash-link" href="https://crawlee.dev/blog/crawlee-for-python-v1#robots-exclusion-standard" title="Direct link to Robots exclusion standard">​</a></h2>
<p>Respecting <a href="https://en.wikipedia.org/wiki/Robots.txt" rel="noopener noreferrer" target="_blank"><code>robots.txt</code></a> is an important part of responsible web crawling. This simple file lets website owners declare which parts of their site should not be crawled by automated agents. Crawlee v1 makes it trivial to follow these rules: just set the <code>respect_robots_txt_file</code> option on your crawler, and Crawlee will automatically check the file before issuing requests.</p>
<p>This not only helps you build ethical crawlers, but can also save time and bandwidth by skipping disallowed or irrelevant pages. For example, login pages, search results, or admin sections are often excluded in <a href="https://www.en.wikipedia.org/wiki/Robots.txt" rel="noopener noreferrer" target="_blank"><code>robots.txt</code></a>, and Crawlee will handle that for you automatically.</p>

<h2 class="anchor anchorTargetHideOnScrollNavbar_vjPI" id="fingerprinting">Fingerprinting<a class="hash-link" href="https://crawlee.dev/blog/crawlee-for-python-v1#fingerprinting" title="Direct link to Fingerprinting">​</a></h2>
<p>Modern websites often rely on browser fingerprinting to distinguish real users from automated traffic. Instead of just checking the <a href="https://www.developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/User-Agent" rel="noopener noreferrer" target="_blank">User-Agent</a> header, they combine dozens of subtle signals - supported fonts, canvas rendering, WebGL features, media devices, screen resolution, and more. Together, these form a unique <a href="https://www.en.wikipedia.org/wiki/Device_fingerprint" rel="noopener noreferrer" target="_blank">device fingerprint</a> that can easily expose headless browsers or automation frameworks.</p>
<p>Without fingerprinting, Playwright sessions tend to look identical and are more likely to be flagged by anti-bot systems. Crawlee v1 integrates with the <a href="https://www.crawlee.dev/python/api/class/FingerprintGenerator" rel="noopener noreferrer" target="_blank"><code>FingerprintGenerator</code></a> to automatically inject realistic, randomized fingerprints into every <a href="https://www.crawlee.dev/python/api/class/PlaywrightCrawler" rel="noopener noreferrer" target="_blank"><code>PlaywrightCrawler</code></a> session. This modifies HTTP headers, browser APIs, and other low-level signals so that each crawler run looks like a real browser on a real device.</p>
<p>Using fingerprinting in Crawlee is straightforward: create a fingerprint generator with your desired options and pass it to the crawler.</p>

<h2 class="anchor anchorTargetHideOnScrollNavbar_vjPI" id="open-telemetry">Open telemetry<a class="hash-link" href="https://crawlee.dev/blog/crawlee-for-python-v1#open-telemetry" title="Direct link to Open telemetry">​</a></h2>
<p>Running crawlers in production means you often want more than just logs - you need visibility into what the crawler is doing, how it's performing, and where bottlenecks occur. Crawlee v1 adds basic <a href="https://www.opentelemetry.io/" rel="noopener noreferrer" target="_blank">OpenTelemetry</a> instrumentation via <a href="https://www.crawlee.dev/python/api/class/CrawlerInstrumentor" rel="noopener noreferrer" target="_blank"><code>CrawlerInstrumentor</code></a>, giving you a standardized way to collect traces and metrics from your crawlers.</p>
<p>With <a href="https://www.opentelemetry.io/" rel="noopener noreferrer" target="_blank">OpenTelemetry</a> enabled, Crawlee automatically records information such as:</p>
<ul>
<li class="">Requests and responses (including timings, retries, and errors).</li>
<li class="">Resource usage events (memory, concurrency, system snapshots).</li>
<li class="">Lifecycle events from crawlers, routers, and handlers.</li>
</ul>
<p>These signals can be exported to any OpenTelemetry-compatible backend (e.g. <a href="https://www.jaegertracing.io/" rel="noopener noreferrer" target="_blank">Jaeger</a>, <a href="https://www.prometheus.io/" rel="noopener noreferrer" target="_blank">Prometheus</a>, or <a href="https://www.grafana.com/" rel="noopener noreferrer" target="_blank">Grafana</a>), where you can monitor real-time dashboards or analyze traces to understand crawler performance.</p>
