<p><a href="https://bsky.app/" rel="noopener noreferrer" target="_blank">Bluesky</a> is an emerging social network developed by former members of the <a href="https://x.com/" rel="noopener noreferrer" target="_blank">Twitter</a>(now X) development team. The platform has been showing significant growth recently, reaching 140.3 million visits according to <a href="https://www.similarweb.com/website/bsky.app/#traffic" rel="noopener noreferrer" target="_blank">SimilarWeb</a>. Like X, Bluesky generates a vast amount of data that can be used for analysis. In this article, we’ll explore how to collect this data using <a href="https://github.com/apify/crawlee-python" rel="noopener noreferrer" target="_blank">Crawlee for Python</a>.</p>

<h3 class="anchor anchorTargetHideOnScrollNavbar_vjPI" id="3-configuring-crawlee-for-python-for-data-collection">3. Configuring Crawlee for Python for data collection<a class="hash-link" href="https://crawlee.dev/blog/scrape-bluesky-using-python#3-configuring-crawlee-for-python-for-data-collection" title="Direct link to 3. Configuring Crawlee for Python for data collection">​</a></h3>
<p>Since we’ll be using the official API, we do not need to worry about being blocked by Bluesky. However, we should be careful with the number of requests to avoid overloading Bluesky's servers, so we will configure <a href="https://www.crawlee.dev/python/api/class/ConcurrencySettings" rel="noopener noreferrer" target="_blank"><code>ConcurrencySettings</code></a>. We’ll also configure <a href="https://www.crawlee.dev/python/api/class/HttpxHttpClient" rel="noopener noreferrer" target="_blank"><code>HttpxHttpClient</code></a> to use custom headers with the current session's <code>Authorization</code>.</p>
<p>We’ll use 2 endpoints for data collection: <a href="https://docs.bsky.app/docs/api/app-bsky-feed-search-posts" rel="noopener noreferrer" target="_blank">searchPosts</a> for posts and <a href="https://docs.bsky.app/docs/api/app-bsky-actor-get-profile" rel="noopener noreferrer" target="_blank">getProfile</a>. If you plan to scale the crawler, you can use <a href="https://docs.bsky.app/docs/api/app-bsky-actor-get-profiles" rel="noopener noreferrer" target="_blank">getProfiles</a> for user data, but in this case, you’ll need to implement deduplication logic. When each link is unique, Crawlee for Python handles this for you.</p>
<p>When collecting data, I’d like to separately collect user and post data, so we’ll use different <a href="https://www.crawlee.dev/python/api/class/Dataset" rel="noopener noreferrer" target="_blank"><code>Dataset</code></a> instances for storage.</p>
