<h2 class="anchor anchorTargetHideOnScrollNavbar_vjPI" id="2-analyzing-youtube-and-determining-a-scraping-strategy">2. Analyzing YouTube and determining a scraping strategy<a class="hash-link" href="https://crawlee.dev/blog/scrape-youtube-python#2-analyzing-youtube-and-determining-a-scraping-strategy" title="Direct link to 2. Analyzing YouTube and determining a scraping strategy">​</a></h2>
<p>If you're working on a small project to extract data from YouTube, you should use the <a href="https://developers.google.com/youtube/v3/docs/search/list" rel="noopener noreferrer" target="_blank">YouTube API</a> to get your data. However, the API has very strict quotas, with no more than <a href="https://developers.google.com/youtube/v3/determine_quota_cost" rel="noopener noreferrer" target="_blank">10,000 units per day</a>. This allows you to get just 100 search pages, and you can't increase this limit.</p>
<p>If your project requires more data than the API allows, you'll need to use crawling. Let's examine the site to develop an optimal crawling strategy.</p>
<p>Let's study YouTube navigation using <a href="https://www.youtube.com/@Apify" rel="noopener noreferrer" target="_blank">Apify's YouTube channel</a> as an example to better understand the features and data extraction points.</p>
<p>YouTube uses infinite scrolling to load new elements on the page, similar to what we discussed in the corresponding <a href="https://www.crawlee.dev/blog/infinite-scroll-using-python" rel="noopener noreferrer" target="_blank">article</a> from the <a href="https://apify.com/" rel="noopener noreferrer" target="_blank">Apify</a> team. Let's look at how this works using <a href="https://developer.chrome.com/docs/devtools" rel="noopener noreferrer" target="_blank">DevTools</a> and the <a href="https://developer.chrome.com/docs/devtools/network/" rel="noopener noreferrer" target="_blank">Network</a> tab.</p>
<p><img alt="Load Request" class="img_ev3q" height="1066" src="https://crawlee.dev/assets/images/load_request-c583830dda107ae55fb6426d7b96e569.webp" width="1917" /></p>
<p>If we look at the response structure, we can see that YouTube uses <a href="https://www.json.org/" rel="noopener noreferrer" target="_blank">JSON</a> to transmit data, but its structure is quite complex to navigate.</p>
<p><img alt="Load Response" class="img_ev3q" height="1064" src="https://crawlee.dev/assets/images/load_response-7061bb91cadc904d54073c033f3f0a20.webp" width="1919" /></p>
<p>Therefore, we'll use <a href="https://playwright.dev/python/docs/intro" rel="noopener noreferrer" target="_blank">Playwright</a> for crawling, which will help us avoid parsing complex JSON responses. But if you want to practice crawling complex websites, try implementing a crawler based on an HTTP client, like in this <a href="https://www.crawlee.dev/blog/scraping-dynamic-websites-using-python" rel="noopener noreferrer" target="_blank">article</a>.</p>
<p>Let's analyze the selectors for getting video links using the <a href="https://developer.chrome.com/docs/devtools/elements/" rel="noopener noreferrer" target="_blank">Elements</a> tab:</p>
<p><img alt="Selectors" class="img_ev3q" height="1064" src="https://crawlee.dev/assets/images/selectors-745f5daab12810cc998990e4c066afdf.webp" width="1919" /></p>
<p>It looks like we're interested in <code>a</code> tags with the attribute <code>id="video-title-link"</code>!</p>
<p>Let's look at the video page to understand better how YouTube transmits data. As expected, we see data in JSON format.</p>
<p><img alt="Video Response" class="img_ev3q" height="1061" src="https://crawlee.dev/assets/images/video_json-44affd2ba348740caa8d1bc79ba9a8a9.webp" width="1919" /></p>
<p>Now let's get the transcript link. Click on the subtitles button in the player to trigger the transcript request.</p>
<p><img alt="Transcript Request" class="img_ev3q" height="1074" src="https://crawlee.dev/assets/images/transcript_request-77c78163912afe398161b431c20cb733.webp" width="1917" /></p>
<p>Let's verify that we can access the transcript via this link. Remove the <code>fmt=json3</code> parameter from the URL and open it in your browser. Removing the <code>fmt</code> parameter is necessary to get the data in a convenient XML format instead of the complex JSON3 format.</p>
<p><img alt="Transcript Response" class="img_ev3q" height="1107" src="https://crawlee.dev/assets/images/transcript_response-06133506fa3559a10cfc43912d1af67c.webp" width="1637" /></p>
<p>If you live in a country where <a href="https://gdpr-info.eu/" rel="noopener noreferrer" target="_blank">GDPR</a> applies, you'll need to handle the following pop-up before you can access the data:</p>
<p><img alt="GDPR" class="img_ev3q" height="1071" src="https://crawlee.dev/assets/images/GDPR-103ec4d5f927916f704ec1d4d597bd82.webp" width="1919" /></p>
<p>After our analysis, we now understand:</p>
<ul>
<li class=""><strong>Navigation strategy</strong>: How to navigate the channel page to retrieve all videos using infinite scroll.</li>
<li class=""><strong>Video metadata extraction</strong>: How to extract video statistics, title, description, publish date, and other metadata from video pages.</li>
<li class=""><strong>Transcript access</strong>: How to obtain the correct transcript link.</li>
<li class=""><strong>Data formats</strong>: Transcript data is available in XML format, which is easier to parse than JSON3</li>
<li class=""><strong>Regional considerations</strong>: Special handling required for GDPR consent in European countries</li>
</ul>
<p>With this knowledge, we're ready to implement the YouTube scraper using Crawlee for Python.</p>
<h2 class="anchor anchorTargetHideOnScrollNavbar_vjPI" id="3-configuring-crawlee">3. Configuring Crawlee<a class="hash-link" href="https://crawlee.dev/blog/scrape-youtube-python#3-configuring-crawlee" title="Direct link to 3. Configuring Crawlee">​</a></h2>
<p>Configuring Crawlee for YouTube is very similar to configuring it for <a href="https://www.crawlee.dev/blog/scrape-tiktok-python" rel="noopener noreferrer" target="_blank">TikTok</a>, but with some key differences.</p>
<p>Since pages have infinite scrolling, we need to limit the number of elements we want to get. For this, we'll add a <code>max_items</code> parameter that will limit the maximum number of elements for each search, and pass it in <code>user_data</code> when forming a <a href="https://www.crawlee.dev/python/api/class/Request" rel="noopener noreferrer" target="_blank">Request</a>.</p>
<p>We'll limit the intensity of scraping by setting <code>max_tasks_per_minute</code> in <a href="https://www.crawlee.dev/python/api/class/ConcurrencySettings" rel="noopener noreferrer" target="_blank"><code>ConcurrencySettings</code></a>. This will help us reduce the likelihood of being blocked by YouTube.</p>
<p>Scrolling pages can take a long time, so we’ll increase the time limit for processing a single request using <code>request_handler_timeout</code>.</p>
<p>Since we won't be saving images, videos, and similar media content during crawling, we can block requests to them using <a href="https://www.crawlee.dev/python/api/class/BlockRequestsFunction" rel="noopener noreferrer" target="_blank"><code>block_requests</code></a> and <a href="https://www.crawlee.dev/python/api/class/PlaywrightCrawler#pre_navigation_hook" rel="noopener noreferrer" target="_blank"><code>pre_navigation_hook</code></a>.</p>
<p>Also, to handle the <code>GDPR</code> page only once, we'll use <a href="https://www.crawlee.dev/python/api/class/UseStateFunction" rel="noopener noreferrer" target="_blank"><code>use_state</code></a> to pass the appropriate cookies between sessions, ensuring all requests have the necessary cookies.</p>

<h2 class="anchor anchorTargetHideOnScrollNavbar_vjPI" id="4-extracting-youtube-data">4. Extracting YouTube data<a class="hash-link" href="https://crawlee.dev/blog/scrape-youtube-python#4-extracting-youtube-data" title="Direct link to 4. Extracting YouTube data">​</a></h2>

<p>For infinite scrolling, we'll use the built-in helper function <a href="https://www.crawlee.dev/python/api/class/PlaywrightCrawlingContext#infinite_scroll" rel="noopener noreferrer" target="_blank">'infinite_scroll'</a>.
